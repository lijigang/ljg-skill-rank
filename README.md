# ljg-skill-rank

降秩引擎 — 一个 Claude Code Skill，对任意领域执行"降秩"操作，找到不可再分的独立生成器。

## 什么是"秩"

秩不是"关键要素"，不是"核心原则"，不是"总结要点"。

秩是严格的：**这个领域里，真正独立的生成器有几个？用它们能不能反向生成全部现象？** 能，才算找到了秩。

## 功能

- 从现象出发（不从定义出发），铺开 10-15 个典型现象
- 提取描述维度，发现维度间的硬约束
- 降维到不可再分的独立生成器（秩 = 维度 - 约束）
- 逐一生成验证：每个现象都必须能被生成器组合解释
- 画出生成器之间的耦合拓扑（ASCII 接线图）
- 输出结构化 org-mode 文件

## 安装

```bash
/plugin marketplace add lijigang/ljg-skill-rank
/plugin install ljg-rank
```

## 使用

```
/ljg-rank 经济学
/ljg-rank 烹饪
降秩：投资
找秩：写作
```

## 输出示例

```org
* 领域：经济学

* 降秩过程

** 维度（7 个）
** 约束（3 条）
** 秩 = 4

生成器：稀缺性、激励响应、时间、信息分布

* 生成验证

| 现象           | 能否生成 | 用了哪些生成器         |
|----------------+----------+------------------------|
| 价格随供需变动 | ✓        | 稀缺性 + 激励 + 信息   |
| ...            | ...      | ...                    |

* 接线图

  {ASCII 拓扑：因果箭头 + 反馈环}

* 一句话

  经济学 = 稀缺、激励、时间、信息四根弦的振动与耦合
```

## 质量红线

- 约束要硬——"好像有关系"不算，能说清"知道 A 推出 B"才算
- 生成测试不能跳过——跳过它，秩就只是感觉，不是真相
- 秩宜少不宜多——超过 5 大概率约束没找够

## 设计原则

- 基于 Soul.md 中"理解即降秩"的完整回路
- 从现象出发 → 维度（外赋）→ 秩（内禀）→ 结构（耦合拓扑）→ 反向生成验证
- 秩只是数量，接线图才是结构——再生能力住在接线图里
